Week 6 Workshop Assignment:Capstone Project Instructions

Power BI Dashboard

Author

SI Analytics

Introduction

Welcome to the final project of the Accelerated Data Analytics Bootcamp! For this project, you will build a multi-page dashboard that explores and visualizes real-world data from Gapminder. This is an opportunity to consolidate your skills in data manipulation, visualization and dashboard creation, while creating a portfolio-worthy project that you can share with future employers.

Project Requirements

Data Selection

Your primary dataset for this project should be the cleaned and analysis-ready outputs that you exported during your Week 5 assignment.

These exported datasets have already been cleaned, transformed, and prepared in R, and should now be used to build a professional Power BI dashboard.

If you wish to use a different dataset, you may instead:

  • use two or more indicators from the Gapminder’s data repository; or
  • propose another dataset that is more relevant to your work.

Alternative datasets must be approved by emailing: contact@sianalytics.org

Dashboard Structure

Your dashboard should contain at least THREE report pages.

At a minimum, include:

  1. Indicator One Overview
  2. Indicator Two Overview
  3. Comparative Insights (showing the relationship between the two indicators)

Additional pages are encouraged where appropriate.

Technical Requirements

Your dashboard should include at least 12 report elements.

Examples include:

  • KPI Cards
  • Bar Charts
  • Column Charts
  • Line Charts
  • Scatter Plots
  • Pie or Donut Charts
  • Filled or Bubble Maps
  • Tables
  • Matrix Tables
  • Slicers
  • Buttons or Navigation
  • Images or Icons
  • Text Boxes summarising key findings

You may combine these in any way provided your dashboard remains clear and uncluttered.

Each report page should contain at least one written interpretation summarising the key insights shown on that page.

Remember:

Good dashboards do not merely display data.

They communicate evidence that supports better decision-making.

Submission

Export your completed dashboard to PDF from Power BI Desktop.

Submit:

  • Power BI (.pbix) file
  • Exported PDF
  • Any supporting datasets used

Grading Rubric (100 points)

Data Selection and Manipulation (25 points)

  • Relevant data selected from Gapminder or approved alternative source
  • Data files are properly cleaned without errors or inconsistencies
  • Datasets are correctly merged when needed
  • Shows evidence that data were professionally prepared in R using dplyr before being imported into Power BI.

Dashboard Structure (20 points)

  • Contains required three pages (first indicator, second indicator, relationship)
  • Layout is logical, easy to navigate, and makes effective use of space
  • Design choices enhance understanding of the content

Visualization Elements (25 points)

  • Dashboard includes at least 12 display elements (an element is a KPI boxes, filters, plot, table, map, etc.)
  • Plots have clear titles, labels, and appropriate scales
  • Interactive features (e.g. hover-over text) function correctly and display clear and useful information
  • Visual elements have titles or text boxes that explain the key insights
  • Appropriate choice of visualization types for the data
  • Consistent formatting and style

Originality, Creativity, and Insight (10 points)

  • Original approaches to dashboard design, visualization, or interaction
  • Demonstrates thoughtful exploration of the data
  • Presents interesting patterns or relationships

Professional Dashboard Design (10 points)

The dashboard demonstrates:

  • consistent colour palette
  • appropriate typography
  • logical spacing
  • alignment
  • minimal clutter
  • professional visual hierarchy

The dashboard should encourage exploration through appropriate interactivity, such as:

  • slicers
  • drill-through
  • tooltips
  • page navigation
  • bookmarks (optional)
  • cross-filtering

Think Like an Analyst

As you build your dashboard, ask yourself:

  • What story does this data tell?
  • What questions should a decision-maker be able to answer?
  • Which visual communicates this insight most effectively?
  • What additional context would improve interpretation?
  • If someone viewed this dashboard for the first time, would they immediately understand the key findings?

Remember that the goal is not simply to create attractive visuals, but to communicate meaningful insights that support evidence-based decisions.

Final Reminder

This project represents the culmination of everything you have learned throughout the bootcamp.

Treat it as a professional portfolio piece.

Aim for a dashboard that you would be confident presenting during a job interview or sharing with a prospective employer. Focus on clarity, professionalism, usability, and meaningful insight.
